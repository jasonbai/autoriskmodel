# AutoResearch 智能风控报告生成器

## 功能概述

生成Markdown格式的模型评估报告，包含完整的模型性能分析、评分分布、特征重要性等信息。

## 使用方法

### 基础用法

```bash
# 使用最新模型生成报告
python reporter.py

# 指定模型文件
python reporter.py --model data/cache/models/lightgbm_20260320_101241.pkl

# 指定输出路径
python reporter.py --output my_report.md
```

说明：
- `--output` 传相对路径时，文件会写入 `report/` 目录
- 如果希望输出到指定绝对路径，请传绝对路径

## 报告内容

生成的Markdown报告包含以下章节：

| 章节 | 内容 |
|------|------|
| **数据概览** | 样本数、坏样本率、特征数量 |
| **模型性能** | AUC、KS、过拟合程度 |
| **模型稳定性** | PSI分析、稳定性评级 |
| **评分分布** | 等频分箱统计、逾期率、Lift |
| **特征重要性** | Top 20特征排序 |
| **模型参数** | 模型超参数列表 |
| **总结** | 性能评价、改进建议 |

## 评估指标说明

### AUC (Area Under ROC Curve)

**含义**: 模型排序能力

**范围**: 0.5 (随机) ~ 1.0 (完美)

**评判标准**:
- > 0.70: 优秀
- 0.65-0.70: 良好
- 0.60-0.65: 一般
- < 0.60: 需改进

### KS (Kolmogorov-Smirnov)

**含义**: 模型区分度

**范围**: 0 ~ 1.0

**评判标准**:
- > 0.30: 优秀
- 0.20-0.30: 良好
- < 0.20: 较弱

### PSI (Population Stability Index)

**含义**: 模型稳定性

**评判标准**:
- < 0.1: 优秀 ✅
- 0.1-0.2: 良好 ⚠️
- > 0.2: 需要关注 ❌

### 过拟合程度

**含义**: train_auc - oot_auc

**目标**: 越小越好

**评判标准**:
- < 0.03: 优秀
- 0.03-0.05: 良好
- > 0.05: 需要关注

## 常见问题

### Q: 报告中的AUC很低（<0.6），怎么办？

**A:** 可能原因：
1. 特征质量不足
2. 需要调参
3. 数据泄露未处理

**建议**:
- 使用 AutoResearch 进行自动研究
- 尝试启用 `extra_trees=True`
- 检查特征工程

### Q: PSI很高（>0.2），怎么办？

**A:** 说明模型不稳定：
1. 降低模型复杂度
2. 增加正则化
3. 使用 `extra_trees=True`
4. 减少 `feature_fraction`

### Q: 过拟合严重（>0.1），怎么办？

**A:** 建议调整：
1. 减少 `num_leaves`
2. 降低 `max_depth`
3. 增加 `reg_alpha` 和 `reg_lambda`
4. 启用 `extra_trees=True`
5. 减少 `n_estimators`

### Q: 如何查看生成的报告？

**A:**
- 使用Markdown阅读器
- 使用文本编辑器
- 在GitHub/GitLab上直接预览
