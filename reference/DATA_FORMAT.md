# 数据格式说明

## 实验数据集要求

### 必需列

| 列名 | 类型 | 说明 | 示例 |
|------|------|------|------|
| `y_flag` | int | 目标变量（0=好样本，1=坏样本） | 0, 1 |
| `window_flag` | string | 数据集分割标识 | train, val, oot |

### 特征列

- **类型**: 数值型（int 或 float）
- **数量**: 建议 50-500 个特征
- **要求**: 特征列名不能包含以下模式：
  - `mob` - Mobile on Book（未来账期信息）
  - `fpd` - First Payment Default（首次违约）
  - `dpd` - Days Past Due（逾期天数）

### 自动删除的列

以下列会被自动删除，无需手动清理：

1. **标识符列**: `appl_seq`, `apply_dt`, `rptno`, `id_unqf`, `id_unqp` 等
2. **文本列**: 所有 `object` 类型的列
3. **数据泄露特征**: 包含 `mob`, `fpd`, `dpd` 的列（目标变量除外）

## 数据集分割

根据 `window_flag` 列自动分割：

| window_flag | 数据集 | 用途 | 建议比例 |
|-------------|--------|------|----------|
| `train` | 训练集 | 模型拟合 | 70-80% |
| `val` | 验证集 | 参数调优 | 10-15% |
| `oot` | 测试集 | 泛化测试 | 10-15% |

## 最小数据示例

```csv
y_flag,window_flag,feature_1,feature_2,feature_3
0,train,0.5,1.2,0.8
1,train,0.3,0.9,1.1
0,train,0.7,1.5,0.6
0,val,0.4,1.1,0.9
1,val,0.6,0.8,1.2
0,oot,0.5,1.3,0.7
1,oot,0.2,1.0,1.0
```

## 数据质量要求

| 指标 | 要求 | 说明 |
|------|------|------|
| **样本量** | >10,000 | 样本越多模型越好 |
| **坏样本率** | 1-20% | 目标变量中 1 的比例 |
| **缺失值** | <95% | 单个特征的缺失值比例 |
| **特征数** | 10-500 | 过少欠拟合，过多过拟合 |

## 准备步骤

1. 准备 CSV 文件，确保包含 `y_flag` 和 `window_flag` 列
2. 将文件放置在 `reference/train.csv`
3. 运行数据准备：
   ```bash
   python prepare.py reference/train.csv
   ```
4. 开始训练：
   ```bash
   python train.py
   ```
